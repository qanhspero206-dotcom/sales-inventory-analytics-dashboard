# Sales & Inventory Analytics Dashboard for Mini Mart

## Project Overview

This beginner-friendly data analytics project analyzes sales and inventory data for a mini mart. The dashboard helps identify best-selling products, stockout-risk products, slow-moving inventory, and practical restocking recommendations.

The project demonstrates how raw CSV data can be turned into business insights using Python and Streamlit.

## Business Problem

Mini mart managers need to keep popular products available while avoiding too much cash tied up in slow-moving inventory. Without a simple data view, restocking decisions may rely only on experience.

This project answers four business questions:

1. Which products sell the most units?
2. Which categories generate the most revenue?
3. Which products are at risk of running out of stock?
4. Which products may be slow-moving and should be reviewed before reordering?

## Dataset Description

The dataset contains 50 sample sales and inventory records from 5 mini mart stores over 10 days.

Dataset path: `data/mini_mart_sales_inventory.csv`

| Column | Description |
|---|---|
| `date` | Sales and inventory record date |
| `store_id` | Store identifier |
| `product_id` | Product identifier |
| `product_name` | Product name |
| `category` | Product category |
| `units_sold` | Number of units sold |
| `revenue` | Revenue generated by the product |
| `current_stock` | Stock remaining at the end of the day |
| `reorder_level` | Minimum stock level before replenishment is needed |
| `promotion_status` | Whether the product was promoted |

## Methodology

1. Load the CSV dataset with Python.
2. Validate numeric fields such as revenue, units sold, current stock, and reorder level.
3. Create calculated inventory columns:
   - `stock_status`
   - `reorder_recommendation`
4. Aggregate sales by product and category.
5. Identify stockout-risk products where `current_stock <= reorder_level`.
6. Identify slow-moving products where sales are low and stock remains high.
7. Build an interactive Streamlit dashboard with filters, KPIs, charts, and action tables.

## Key Insights

Based on the sample dataset:

- Total revenue is `$1,493.20` and total units sold are `1,197`.
- Bottled Water 500ml is the top-selling product with `222` units sold.
- Beverages generate the highest category revenue with `$364.10`.
- Several products are at stockout risk because current stock is at or below the reorder level.
- Products with low sales and high remaining stock should be reviewed before more inventory is ordered.

## Recommendations

- Reorder stockout-risk products immediately, especially fast-moving beverage and snack items.
- Monitor popular products during promotions because demand may increase quickly.
- Review slow-moving products before reordering to reduce overstock and free shelf space.
- Use the dashboard regularly so restocking decisions are based on current sales and inventory data.

## How to Run the Streamlit Dashboard

1. Clone or download this repository.
2. Open a terminal in the project folder.
3. Install the required packages:

```bash
pip install -r requirements.txt
```

4. Start the dashboard:

```bash
streamlit run app.py
```

5. Open the local Streamlit URL shown in the terminal.
